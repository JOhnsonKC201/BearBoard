  Last Updated: April 15, 2026
  Reference: See landing-mockup.html for the target design
==============================================================================

  This file assigns each developer's Sprint 1 work. The goal is to build
  toward the design shown in the HTML mockup — editorial layout, sharp
  corners, navy/gold palette, Archivo + Libre Franklin fonts, sidebar
  with trending/events/groups, post cards with left-border accents,
  sort/filter tabs, and a BearBoard Assistant chat widget.

  DESIGN PRINCIPLES (from the mockup):
  - Flat and editorial. No rounded corners — use sharp or 2-4px radius max.
  - Left-border accent on post cards (3px, turns gold on hover).
  - Navy (#0B1D34) and gold (#D4962A) are the primary colors.
  - Fonts: Archivo (headings, labels, weights 700-900) and
    Libre Franklin (body text, weight 400-600).
  - No shadows. Use 1px borders in lightgray (#D5D1CA) for separation.
  - Dense, information-rich layout. No unnecessary whitespace.

  DO NOT (for everyone):
  - Do NOT install extra UI libraries (Material UI, Chakra, etc.).
    We are using Tailwind CSS only.
  - Do NOT change the project folder structure without team agreement.
  - Do NOT push directly to main. Use feature branches and pull requests.

==============================================================================
  GENERAL NOTES
==============================================================================

  - Backend runs with: cd backend && uvicorn main:app --reload
  - Frontend runs with: cd frontend && npm install && npm run dev
  - Backend API docs: http://localhost:8000/docs (Swagger UI)
  - Frontend dev server: http://localhost:3000

  - The backend uses SQLAlchemy + FastAPI. The frontend uses React 18 +
    Vite + Tailwind CSS + React Router.

  - The HTML mockup (landing-mockup.html) is the design target. It shows
    the finished look and feel including colors, typography, spacing,
    component styles, and interactions. Use it as your north star.

  - Coordinate on API contracts before building. Agree on request/response
    shapes in the team chat before coding endpoints.

  - Every feature branch needs a PR. Every PR needs at least one review.

  API ENDPOINTS SUMMARY:
    POST   /api/auth/register     — Register new user
    POST   /api/auth/login        — Login, returns JWT
    GET    /api/auth/me           — Current user from token
    GET    /api/users/{id}        — Get user profile
    PUT    /api/users/{id}        — Update own profile
    GET    /api/posts             — List posts (?sort=&category=&limit=&offset=)
    POST   /api/posts             — Create post (auth required)
    GET    /api/posts/{id}        — Single post with comments
    DELETE /api/posts/{id}        — Delete own post (auth required)
    POST   /api/posts/{id}/vote   — Vote on post (auth required)
    GET    /api/trending          — Top 3 trending posts
    GET    /api/events            — Upcoming events
    POST   /api/chat              — Chat assistant message
